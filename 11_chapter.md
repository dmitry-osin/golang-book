### Глава 11. Потоки и файлы

#### 1. Операции ввода-вывода. Абстракции `io.Reader` и `io.Writer`
Сердце любой системы ввода-вывода (I/O) в Go — это пакет `io`. Он описывает, как данные должны "перетекать" из одного места в другое.



Вся магия строится на двух интерфейсах:
```go
// Reader читает данные из источника (файл, сеть, консоль) и кладет их в ваш срез байтов (p).
type Reader interface {
    Read(p []byte) (n int, err error)
}

// Writer берет данные из вашего среза байтов (p) и пишет их в приемник.
type Writer interface {
    Write(p []byte) (n int, err error)
}
```

*Пример из жизни:* Представь, что тебе нужно перекачать бассейн воды (10 тонн). 
Плохой подход: попытаться поднять весь бассейн целиком (загрузить весь файл в оперативную память). Память компьютера просто лопнет.
Хороший подход (потоковый): взять небольшое ведро (срез байтов, например, на 32 КБ) и черпать воду порциями. Интерфейс `Reader` — это кран, из которого льется вода, а `Writer` — это труба, куда мы ее сливаем.

#### 2. Простые операции: Чтение и запись файлов целиком
Если файл маленький (например, конфиг на пару килобайт), нам не обязательно возиться с потоками. Можно прочитать его целиком одной удобной функцией из пакета `os` (ранее это был пакет `io/ioutil`).

```go
package main

import (
    "fmt"
    "os"
)

func main() {
    // 1. Быстрая запись (создаст файл или перезапишет, если он есть)
    // 0644 - это права доступа в Linux (чтение и запись для владельца, чтение для остальных)
    err := os.WriteFile("hello.txt", []byte("Привет, мир!"), 0644)
    if err != nil {
        panic(err)
    }

    // 2. Быстрое чтение всего файла целиком в память
    data, err := os.ReadFile("hello.txt")
    if err != nil {
        panic(err)
    }
    
    // data - это срез байтов []byte, поэтому конвертируем в строку
    fmt.Println(string(data)) 
}
```

#### 3. Потоковая работа с файлами
Для файлов размером в гигабайты мы должны работать с ними как с потоками.
Функция `os.Open` возвращает указатель на файл (`*os.File`). И самое главное: **`*os.File` автоматически реализует интерфейсы `io.Reader` и `io.Writer`**!

```go
func streamFile() {
    // os.Create создает файл для записи (или очищает существующий)
    file, err := os.Create("log.txt")
    if err != nil {
        fmt.Println("Ошибка:", err)
        return
    }
    defer file.Close() // Обязательно закрываем файл при выходе!

    // Пишем строку (превращая ее в байты)
    file.Write([]byte("Первая строчка лога\n"))
    // Или используем удобную обертку для строк:
    file.WriteString("Вторая строчка лога\n")
}
```

#### 4. Стандартные потоки ввода-вывода и магия `io.Copy`
В любой операционной системе запущенная программа имеет три стандартных потока:
* `os.Stdin` (Стандартный ввод — обычно клавиатура) — это `io.Reader`.
* `os.Stdout` (Стандартный вывод — экран терминала) — это `io.Writer`.
* `os.Stderr` (Стандартный вывод ошибок) — тоже `io.Writer`.

Поскольку и файл, и экран терминала являются `Writer`, мы можем легко перенаправлять данные. Функция `io.Copy(куда, откуда)` берет `Reader` и перекачивает его в `Writer` маленькими порциями под капотом (используя буфер в 32 КБ), не перегружая оперативную память.

```go
import (
    "io"
    "os"
)

func copyExample() {
    file, _ := os.Open("log.txt")
    defer file.Close()

    // Берем данные из файла (Reader) и перекачиваем их прямиком на экран консоли (Writer).
    // Мы можем так вывести на экран файл размером 100 ГБ, потратив всего 32 КБ RAM!
    io.Copy(os.Stdout, file) 
}
```

#### 5. Чтение с консоли и форматируемый ввод
Если мы пишем CLI-утилиту (консольное приложение), нам нужно получать команды от пользователя. Для простых задач используется пакет `fmt`.

```go
func scanExample() {
    var name string
    var age int

    fmt.Print("Введите ваше имя и возраст через пробел: ")
    
    // fmt.Scan читает из os.Stdin, разбивает текст по пробелам 
    // и раскладывает по переданным указателям
    _, err := fmt.Scan(&name, &age)
    if err != nil {
        fmt.Println("Ошибка ввода:", err)
        return
    }

    fmt.Printf("Привет, %s! Тебе %d лет.\n", name, age)
}
```

#### 6. Буферизированный ввод-вывод (Пакет `bufio`)
Функция `fmt.Scan` плохо работает с текстом, в котором есть пробелы (например, если пользователь введет "Иванов Иван", функция прочитает только "Иванов"). К тому же, прямое чтение/запись в файл по одному байту — это очень медленно из-за постоянных "системных вызовов" (system calls) к ядру операционной системы.

*Пример из жизни:* Читать файл по байту — это как ходить в супермаркет за каждой отдельной картофелиной. Буферизация (`bufio`) — это когда ты берешь корзинку (буфер в оперативной памяти). Ты идешь в супермаркет один раз, набиваешь корзинку картошкой (забираешь большой кусок данных из файла), а потом дома достаешь картошку по одной штуке очень быстро.

**Чтение файла построчно:**
```go
import (
    "bufio"
    "fmt"
    "os"
)

func readLines() {
    file, _ := os.Open("big_data.csv")
    defer file.Close()

    // Создаем "сканер" с корзинкой для файла
    scanner := bufio.NewScanner(file)

    // scanner.Scan() читает данные до символа переноса строки (\n).
    // Возвращает true, пока есть что читать.
    for scanner.Scan() {
        text := scanner.Text() // Получаем прочитанную строку
        fmt.Println("Прочитана строка:", text)
    }

    // Обязательно проверяем, не прервалось ли чтение из-за ошибки (например, диск отвалился)
    if err := scanner.Err(); err != nil {
        fmt.Println("Ошибка чтения:", err)
    }
}
```

**Буферизированная запись:**
```go
func bufferedWrite() {
    file, _ := os.Create("fast_log.txt")
    defer file.Close()

    writer := bufio.NewWriter(file) // Создаем корзинку для записи

    writer.WriteString("Эта строка ложится в буфер в оперативной памяти.\n")
    writer.WriteString("В физический файл на диске она еще не попала!\n")

    // Когда мы закончили работу, мы ОБЯЗАНЫ вызвать Flush(), 
    // чтобы принудительно высыпать всё содержимое корзинки на жесткий диск.
    writer.Flush() 
}
```
