### Глава 4. Указатели

#### 1. Что такое указатели
Каждая переменная в программе физически располагается в оперативной памяти компьютера, и у этой ячейки памяти есть свой уникальный номер — адрес (обычно записывается в шестнадцатеричном формате, например `0xc000012088`).

**Указатель (Pointer)** — это просто переменная, которая хранит не сами данные (число или строку), а **адрес** в памяти, где эти данные лежат.

*Пример из жизни:* Представь, что у тебя есть банковская ячейка с деньгами (это переменная со значением). Указатель — это клочок бумаги, на котором написан номер этой ячейки. Сам клочок бумаги ничего не стоит, но он дает прямой доступ к деньгам.



В Go для работы с указателями используются два основных оператора:
* `&` (Амперсанд) — **взятие адреса**. Означает "дай мне адрес переменной".
* `*` (Звездочка) — **разыменование**. Означает "сходи по этому адресу и дай мне/измени само значение".

```go
package main

import "fmt"

func main() {
    age := 30            // Обычная переменная типа int
    
    // Создаем указатель на age. Тип переменной p - *int (указатель на int)
    var p *int = &age    

    fmt.Println("Значение age:", age)       // Выведет: 30
    fmt.Println("Адрес age (указатель):", p) // Выведет что-то вроде 0xc0000a2000

    // Разыменование: читаем значение по адресу
    fmt.Println("Значение по адресу p:", *p) // Выведет: 30

    // Разыменование: изменяем значение по адресу
    *p = 31 // Мы берем клочок бумаги, идем к ячейке и меняем там содержимое
    
    // Оригинальная переменная тоже изменилась, так как мы меняли данные по её адресу!
    fmt.Println("Новое значение age:", age)  // Выведет: 31
}
```

#### 2. Указатели и функции
Как мы разбирали в прошлой главе, в Go параметры передаются в функции **строго по значению**. Это значит, что функция всегда работает с копией данных. Если передать в функцию тяжелый конфиг из 50 полей, Go молча скопирует все 50 полей в новую область памяти. 

Если нам нужно, чтобы функция могла изменить исходную переменную, мы передаем в нее не само значение, а указатель (копируется только "клочок бумаги" с адресом, что происходит мгновенно).

```go
// Функция принимает значение (копию). Изменения не повлияют на оригинал.
func doubleValue(n int) {
    n = n * 2
}

// Функция принимает указатель. Она пойдет по адресу и изменит оригинал.
func doublePointer(n *int) {
    *n = *n * 2
}

func main() {
    score := 100

    doubleValue(score)
    fmt.Println("После doubleValue:", score) // Выведет: 100 (оригинал не изменился)

    // Передаем адрес переменной с помощью &
    doublePointer(&score)
    fmt.Println("После doublePointer:", score) // Выведет: 200 (оригинал изменен)
}
```

#### 3. Указатели против значений: влияние на производительность (Стек vs Куча)
Это ключевая концепция для разработки систем с низкой задержкой (low-latency) и минимизации пауз сборщика мусора (GC).

Память работающего приложения делится на две основные зоны:
1. **Стек (Stack):** Очень быстрая память. Каждая горутина (функция) имеет свой стек. Переменные кладутся в стек при входе в функцию и мгновенно уничтожаются при выходе из нее. Это ничего не стоит сборщику мусора — память очищается автоматически сдвигом указателя стека.
2. **Куча (Heap):** Глобальная, медленная память. Сюда попадают переменные, которые должны "жить" дольше, чем функция, в которой они были созданы. Именно кучу вынужден постоянно сканировать Garbage Collector (GC), чтобы найти мусор и удалить его, что потребляет ресурсы CPU.



**Escape Analysis (Анализ утечек)** — это встроенный в компилятор Go механизм, который решает, куда положить переменную.
* Если ты создаешь переменную внутри функции и передаешь её в другие функции *по значению* (копируя), компилятор гарантирует, что переменная останется в быстром **Стеке**.
* Если ты создаешь переменную, берешь от нее указатель `&` и возвращаешь его наружу (или передаешь в глобальную структуру), компилятор понимает: "Ага, эта переменная нужна кому-то еще после завершения функции". Он делает так называемый *escape to heap* — отправляет переменную в **Кучу**, добавляя работы сборщику мусора.

**Золотое правило Go при выборе между указателем и значением:**
* По умолчанию используй **значения** (копии). Передача структуры по значению часто работает быстрее, чем по указателю, так как она не сбегает в кучу и не нагружает GC. Современные процессоры копируют небольшие куски памяти невероятно быстро.
* Используй **указатели**, только если:
    1. Тебе действительно нужно изменить состояние (мутировать) исходной переменной.
    2. Структура данных настолько огромна (сотни байт/мегабайты), что само её копирование будет стоить дороже, чем последующая работа GC по очистке кучи.
    3. Тип данных логически не может быть скопирован (например, мьютексы `sync.Mutex` или пулы соединений с БД).
