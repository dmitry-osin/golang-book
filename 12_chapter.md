### Глава 12. Сетевое программирование и Микросервисы

#### 1. Низкоуровневая сеть: TCP Сервер и Обработка подключений
HTTP работает поверх протокола TCP. Прежде чем писать REST API, давай посмотрим, как работает "голое" сетевое взаимодействие.

*Пример из жизни:* TCP-соединение — это телефонный звонок. Мы устанавливаем связь, говорим "Алло" (Handshake), а затем начинаем непрерывно обмениваться данными, пока кто-то не положит трубку.

Главная суперсила Go в сети: **"Один коннект — одна горутина"**. Нам не нужно писать сложные асинхронные циклы событий (Event Loops). Сервер просто ждет клиента, и как только клиент звонит, сервер передает трубку отдельной горутине и идет ждать следующего.

```go
package main

import (
    "bufio"
    "fmt"
    "net"
)

func handleConnection(conn net.Conn) {
    // Не забываем закрыть соединение (положить трубку), когда клиент уйдет
    defer conn.Close() 
    fmt.Println("Подключился новый клиент:", conn.RemoteAddr())

    // conn реализует интерфейсы io.Reader и io.Writer!
    // Поэтому мы можем использовать наш удобный сканер-корзинку из прошлой главы.
    scanner := bufio.NewScanner(conn)
    for scanner.Scan() {
        msg := scanner.Text()
        fmt.Println("Получено:", msg)
        
        // Отправляем ответ клиенту (Эхо-сервер)
        conn.Write([]byte("Сервер получил: " + msg + "\n"))
    }
}

func main() {
    // Начинаем слушать TCP порт 8080
    listener, err := net.Listen("tcp", ":8080")
    if err != nil {
        panic(err)
    }
    defer listener.Close()
    fmt.Println("TCP сервер запущен на порту 8080...")

    // Бесконечный цикл ожидания клиентов
    for {
        conn, err := listener.Accept() // Блокируется, пока кто-то не подключится
        if err != nil {
            continue
        }
        // Запускаем обработку клиента в легковесной горутине и идем ждать следующего!
        go handleConnection(conn) 
    }
}
```

#### 2. Отправка запросов (HTTP Клиент) и Установка таймаута
Отправить запрос к чужому API в Go очень просто. Но здесь кроется **самая знаменитая ловушка для новичков**.

В стандартном пакете есть функция `http.Get("url")`. **Никогда не используй её в production-коде!** Она использует дефолтный `http.Client`, у которого *нет таймаутов*. Если чужой сервер зависнет и перестанет отвечать, твой клиент будет ждать вечно, горутины начнут скапливаться, и твой сервер в итоге упадет от нехватки памяти (Memory Leak).

*Правило эксперта:* Всегда создавай свой `http.Client` и жестко задавай таймауты.

```go
import (
    "io"
    "net/http"
    "time"
    "fmt"
)

func fetchWeather() {
    // 1. Создаем правильного клиента
    client := &http.Client{
        Timeout: 5 * time.Second, // Если ответ не придет за 5 секунд - обрываем соединение
    }

    // 2. Создаем запрос
    req, _ := http.NewRequest(http.MethodGet, "https://api.weather.com/current", nil)
    
    // В реальной жизни сюда же добавляются заголовки
    req.Header.Add("Authorization", "Bearer my-token")

    // 3. Выполняем запрос
    resp, err := client.Do(req)
    if err != nil {
        fmt.Println("Ошибка сети или таймаут:", err)
        return
    }
    // ВАЖНО: Обязательно закрываем тело ответа (это сетевой сокет), иначе утечет память!
    defer resp.Body.Close()

    // resp.Body - это снова io.Reader! Читаем его.
    body, _ := io.ReadAll(resp.Body) 
    fmt.Println("Ответ:", string(body))
}
```

#### 3. Написание REST API Сервера (Роутеры и Обработчики)
Теперь напишем полноценный веб-сервер. В Go обработчик HTTP-запроса (Handler) — это любая функция, которая принимает два параметра:
1. `http.ResponseWriter` — куда писать ответ (Writer).
2. `*http.Request` — указатель на структуру с данными запроса (Reader).

```go
import (
    "encoding/json"
    "net/http"
)

type User struct {
    ID   int    `json:"id"`
    Name string `json:"name"`
}

// Обработчик запроса
func getUserHandler(w http.ResponseWriter, r *http.Request) {
    // Проверяем метод
    if r.Method != http.MethodGet {
        http.Error(w, "Метод не разрешен", http.StatusMethodNotAllowed)
        return
    }

    user := User{ID: 1, Name: "Дмитрий"}
    
    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(http.StatusOK) // Отправляем статус 200
    
    // Пакет json умеет писать данные напрямую в Writer (w)
    json.NewEncoder(w).Encode(user) 
}

func main() {
    // Настраиваем роутер (маршрутизатор)
    mux := http.NewServeMux()
    mux.HandleFunc("/api/user", getUserHandler)

    // Запускаем сервер
    http.ListenAndServe(":8080", mux)
}
```
*Примечание:* Начиная с Go 1.22, стандартный роутер `ServeMux` стал очень мощным, научился парсить параметры пути (например, `GET /users/{id}`) и методы, поэтому потребность в сторонних роутерах (типа `chi` или `gorilla/mux`) сильно снизилась.

#### 4. Паттерн Middleware (Промежуточное ПО)
Если нам нужно проверять авторизацию или логировать запросы, писать этот код в каждом обработчике — нарушение принципа DRY (Don't Repeat Yourself). Мы используем *Middleware* — функции, которые оборачивают другие функции, как матрешка.

*Пример из жизни:* Middleware — это рамка металлодетектора перед входом на концерт (обработчик). Сначала ты проходишь проверку, и только если всё ок, тебя пускают внутрь.

```go
// Middleware принимает обработчик и возвращает новый обработчик
func logMiddleware(next http.HandlerFunc) http.HandlerFunc {
    return func(w http.ResponseWriter, r *http.Request) {
        fmt.Println("Входящий запрос:", r.Method, r.URL.Path)
        
        // Передаем управление реальному обработчику
        next(w, r) 
    }
}

// Использование:
// mux.HandleFunc("/api/user", logMiddleware(getUserHandler))
```

#### 5. Микросервисное взаимодействие: gRPC и Protocol Buffers
REST API и JSON отлично подходят для общения между браузером (фронтендом) и сервером. Но когда два твоих микросервиса общаются друг с другом внутри кластера, JSON становится "бутылочным горлышком". Он медленный (текст нужно парсить) и объемный (много лишних кавычек и скобок).

Для Highload-систем стандартом де-факто стал **gRPC** (разработанный Google).



Как это работает:
1. **Protocol Buffers (Protobuf):** Вместо JSON ты пишешь строгий контракт (схему) в специальном файле `.proto`.
   ```protobuf
   // пример файла user.proto
   message UserRequest {
       int32 user_id = 1;
   }
   message UserResponse {
       string name = 1;
   }
   ```
2. **Кодогенерация:** Ты нажимаешь кнопку (вызываешь утилиту `protoc`), и она автоматически генерирует идеальный, сверхбыстрый код на Go для этих структур.
3. **Бинарный формат:** Данные передаются по сети не в виде текста, а сжимаются в бинарный код. Это в разы быстрее и весит в несколько раз меньше, чем JSON.
4. **HTTP/2:** gRPC работает поверх HTTP/2, что позволяет мультиплексировать (передавать множество независимых запросов одновременно) через одно TCP-соединение.

Переход от REST к gRPC для внутренних сервисов часто дает бесплатный буст производительности системы в несколько раз и резко снижает задержки (latency) при каскадных вызовах микросервисов.
