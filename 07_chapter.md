### Глава 7. Строки

#### 1. Введение в строки и руны (Магия UTF-8)
В большинстве старых языков (например, C) один символ всегда равен одному байту (кодировка ASCII), чего хватало только для английского алфавита. В языках вроде Java или C# строка — это массив двухбайтовых символов (UTF-16). 

Go использует более умный подход — **UTF-8**. 
В Go строка (`string`) — это **неизменяемый (read-only) срез байтов**. 
* Английская буква 'A' занимает **1 байт**.
* Русская буква 'А' занимает **2 байта**.
* Эмодзи '🚀' занимает **4 байта**.



Из-за этого возникает главная ловушка для новичков: функция `len()` для строки возвращает не количество *символов*, а количество *байтов*!

Для правильной работы с символами в Go есть специальный тип — **`rune`** (руна). Это просто псевдоним (алиас) для типа `int32` (4 байта). Одна руна гарантированно вмещает в себя любой символ Unicode.

*Пример из жизни:* Представь строку как поезд. В английском языке каждый пассажир (символ) занимает одно кресло (байт). В русском языке пассажиры едут с багажом и занимают по два кресла. Если ты посчитаешь кресла (вызовешь `len()`), ты получишь неправильное количество пассажиров. Чтобы посчитать именно людей, нужно пройтись по вагону и пересчитать их по головам (конвертировать в руны).

```go
package main

import (
    "fmt"
    "unicode/utf8"
)

func main() {
    strEn := "Hello"
    strRu := "Привет"

    // Подсчет байтов
    fmt.Println("Байт в 'Hello':", len(strEn))   // Выведет: 5 (5 букв * 1 байт)
    fmt.Println("Байт в 'Привет':", len(strRu)) // Выведет: 12 (6 букв * 2 байта)

    // Если мы попытаемся взять первую букву по индексу, мы получим не 'П', 
    // а только первый байт от 'П' (какую-то непонятную цифру).
    fmt.Println("Первый байт 'Привет':", strRu[0]) // Выведет: 208

    // Правильный подсчет количества символов (рун)
    fmt.Println("Символов в 'Привет':", utf8.RuneCountInString(strRu)) // Выведет: 6

    // Итерация по строке (for range автоматически декодирует UTF-8 в руны)
    for index, char := range strRu {
        // %c - глагол форматирования для вывода символа
        fmt.Printf("Индекс (байт): %d, Символ (руна): %c\n", index, char)
    }
}
```

#### 2. Пакет `strings`. Операции со строками
Стандартная библиотека Go предоставляет мощный пакет `strings` для любых манипуляций с текстом: поиска, замены, разделения и объединения.

```go
import (
    "fmt"
    "strings"
)

func main() {
    text := "go,rust,zig,java"

    // 1. Разделение строки по разделителю в срез (slice) строк
    langs := strings.Split(text, ",")
    fmt.Println(langs[1]) // Выведет: rust

    // 2. Объединение среза обратно в строку
    joined := strings.Join(langs, " | ")
    fmt.Println(joined) // Выведет: go | rust | zig | java

    // 3. Поиск подстроки
    containsZig := strings.Contains(text, "zig") // Возвращает true

    // 4. Замена (последний аргумент - сколько замен сделать, -1 означает "все")
    replaced := strings.Replace(text, "java", "c++", 1)
    
    // 5. Изменение регистра
    upper := strings.ToUpper("hello") // HELLO
}
```

#### 3. Оптимизация работы со строками: `strings.Builder`
Это критически важный навык. Как уже упоминалось, **строки в Go неизменяемы (immutable)**. Как только строка создана, её байты нельзя изменить.

Что происходит, когда ты складываешь строки через оператор `+` в цикле?
*Пример из жизни:* Представь, что ты строишь башню из Lego. Складывать строки через `+` — это каждый раз, когда тебе нужно добавить один кубик, разрушать старую башню, покупать новую огромную коробку и строить башню заново, перекладывая все старые кубики. Это катастрофически медленно и забивает мусором (старыми версиями башен) всю оперативную память.

**Плохой подход ($O(N^2)$ по памяти и времени):**
```go
func badConcat(words []string) string {
    var result string
    for _, w := range words {
        // При каждой итерации выделяется НОВАЯ память, 
        // туда копируется старый result и добавляется w.
        result += w 
    }
    return result
}
```

**Отличный подход (Zero-Allocation паттерн):**
Для конкатенации в цикле нужно использовать специальную структуру `strings.Builder`. Она выделяет один большой буфер памяти (который можно задать заранее) и просто дописывает в него байты, как в обычный массив, не создавая промежуточных копий.

```go
import "strings"

func goodConcat(words []string) string {
    // 1. Создаем строитель строк
    var builder strings.Builder
    
    // 2. Если мы примерно знаем итоговый размер, мы можем сразу 
    // выделить под него память (capacity), чтобы избежать реалокаций под капотом.
    // Например, резервируем 100 байт.
    builder.Grow(100) 

    for _, w := range words {
        // 3. Записываем строку прямо в буфер
        builder.WriteString(w) 
    }
    
    // 4. Генерируем финальную строку ОДИН раз
    return builder.String() 
}
```
