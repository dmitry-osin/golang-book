### Глава 1. Введение в язык Go

#### Что такое Go
Go (или Golang) — это компилируемый, статически типизированный язык программирования, созданный внутри Google. Его разработали инженеры (Роб Пайк, Кен Томпсон и Роберт Гризмер), которые устали от медленной компиляции и переусложненности других промышленных языков. 

**Пример из жизни:** Представь себе строительство дома. 
* В языках вроде Python или JavaScript есть прораб-переводчик (интерпретатор), который читает чертеж и на ходу объясняет рабочим, что делать. Это гибко, но медленно. 
* В языках вроде Java чертеж переводится в специальный универсальный формат, для чтения которого на стройку нужно привезти огромную, тяжелую машину (виртуальную машину JVM).
* **Go компилируется напрямую в машинный код.** Ты пишешь чертеж, нажимаешь кнопку, и язык выдает тебе готовую бригаду роботов, которые идеально понимают конкретно эту стройплощадку (твою операционную систему и процессор). Им не нужны переводчики или виртуальные машины. Программа получается в виде одного самостоятельного бинарного файла, который работает невероятно быстро.

Главные фишки Go:
1.  **Простота:** Синтаксис минималистичен. В языке всего 25 ключевых слов (для сравнения, в C++ их около 90).
2.  **Скорость:** Компилируется за миллисекунды, работает почти так же быстро, как C/C++.
3.  **Встроенная конкурентность:** Язык изначально создавался для работы на многоядерных процессорах, поэтому выполнять тысячи задач одновременно в нем проще, чем где бы то ни было.

#### Первая программа
Давай напишем классический "Hello, World!" и разберем его по косточкам. 

В Go весь код состоит из пакетов (packages). Каждая программа должна с чего-то начинаться.

```go
// package main указывает, что этот файл является самостоятельной 
// исполняемой программой, а не подключаемой библиотекой.
// Только в пакете main может быть функция main().
package main

// Блок import нужен для подключения других пакетов (встроенных или внешних).
// "fmt" расшифровывается как Format. Это стандартная библиотека для 
// форматированного ввода и вывода (например, текста на экран).
import "fmt"

// func - ключевое слово для объявления функции.
// main - главная функция. Именно с неё начинается выполнение любой Go-программы.
func main() {
    // Вызываем функцию Println из пакета fmt.
    // Она печатает переданную строку в консоль и добавляет перенос на новую строку.
    fmt.Println("Привет, мир!")
}
```

#### Инструментарий языка: `go env`, компилятор и настройка рабочего окружения
В отличие от многих языков, где тулинг (инструменты для работы) нужно собирать из десятков сторонних программ, Go поставляется по принципу "всё включено".

**1. Компилятор и запуск (`go run` и `go build`)**
* Команда `go run main.go` быстро компилирует код во временную папку и сразу его запускает.
    * *Аналогия:* Это как репетиция пьесы. Быстро прочитали по ролям, посмотрели, как звучит, и разошлись. Отлично подходит для этапа разработки.
* Команда `go build main.go` собирает исполняемый бинарный файл (например, `main.exe` на Windows или просто `main` на Linux).
    * *Аналогия:* Это печать тиража книги в типографии. После сборки тебе больше не нужен исходный код (черновики) или сам язык Go на сервере. Ты просто берешь этот файл, копируешь на любой сервер, и он работает сам по себе.

**2. Окружение (`go env`)**
Если в терминале ввести `go env`, язык выдаст огромный список своих настроек. Самые важные для нас:
* `GOOS` (Операционная система) и `GOARCH` (Архитектура процессора). Go умеет делать **кросс-компиляцию**. Сидя на Windows, ты можешь одной командой `GOOS=linux GOARCH=amd64 go build` собрать программу, которая будет идеально работать на Linux-сервере.
* `GOPATH` — историческая папка, где раньше Go хранил весь код. Сейчас используется система модулей (`go mod`), и эта переменная потеряла былую жесткость, став просто местом для кэша и скачанных утилит.

**3. Форматирование кода (`go fmt`)**
В Go нет споров о том, где ставить скобку или использовать табы вместо пробелов. Встроенная утилита `go fmt` автоматически причесывает весь код под единый, жесткий стандарт языка. Ты просто пишешь логику, нажимаешь сохранение, и код выравнивается сам.

**4. Настройка рабочего окружения (LSP и линтеры)**
Для профессиональной и быстрой разработки нет нужды использовать тяжеловесные IDE. Если предпочитаешь скорость и работу без мыши, идеальным вариантом будет модальный редактор вроде NeoVim. 
Достаточно установить менеджер пакетов (например, Mason в сборке LazyVim) и подтянуть два главных инструмента:
* **`gopls`** (Go Language Server) — официальный сервер от Google, который "на лету" анализирует код и передает в твой редактор автодополнение, навигацию к исходникам и ошибки синтаксиса.
* **`golangci-lint`** — мега-линтер, который объединяет под капотом десятки проверок, помогая писать чистый, безопасный и оптимизированный код.
