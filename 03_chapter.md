### Глава 3. Функции

#### 1. Функции и их параметры
Функция в Go — это изолированный блок логики. 
*Пример из жизни:* Представь станок на заводе. У него есть приемный лоток для сырья (параметры) и конвейер для выдачи готовой детали (возвращаемое значение). Станку абсолютно неважно, что происходит в остальном цеху — он берет сырье, делает свою работу и выдает результат.

```go
package main

import "fmt"

// Определение функции:
// func имя_функции(параметры) тип_возвращаемого_значения { ... }
func add(x int, y int) int {
    return x + y
}

// Если параметры имеют одинаковый тип, его можно указать только один раз в конце:
func multiply(x, y, z int) int {
    return x * y * z
}

func main() {
    result := add(5, 7)
    fmt.Println("Сумма:", result)
}
```
**Важно:** В Go параметры всегда передаются **по значению** (копируются). Если ты передашь в функцию огромное число, Go создаст его копию. Если ты попытаешься изменить эту переменную внутри функции, оригинал останется нетронутым. (Исключение составляют ссылочные типы, такие как слайсы и мапы, до которых мы еще доберемся, а также явное использование указателей).

#### 2. Возвращение результата из функции (Множественный возврат)
Это одна из главных фишек Go. В отличие от C или Java, где функция может вернуть только один результат (или сложный объект), в Go функция может возвращать сразу несколько значений. 
Чаще всего это используется для паттерна "значение и ошибка".

```go
// Возвращаем два значения: результат деления и ошибку (в виде строки)
func divide(a, b float64) (float64, string) {
    if b == 0 {
        // Если делим на ноль, возвращаем пустой результат и текст ошибки
        return 0, "ошибка: деление на ноль" 
    }
    // Если всё ок, возвращаем результат и пустую строку
    return a / b, "" 
}

func main() {
    res, err := divide(10, 2)
    if err != "" {
        fmt.Println(err)
    } else {
        fmt.Println("Результат:", res)
    }
    
    // Если нам не нужен один из результатов, мы можем проигнорировать его 
    // с помощью "blank identifier" (пустого идентификатора) - символа подчеркивания '_'
    res2, _ := divide(100, 5) // Игнорируем ошибку (в реальном коде так лучше не делать)
    fmt.Println(res2)
}
```

**Именованные возвращаемые значения:** Ты можешь дать имена результатам прямо в сигнатуре функции. Тогда Go создаст переменные с этими именами и проинициализирует их "нулевыми значениями". Это отличный способ самодокументирования кода.

```go
func rectangleProps(length, width float64) (area float64, perimeter float64) {
    area = length * width
    perimeter = 2 * (length + width)
    // Голый (naked) return. Go сам вернет текущие значения переменных area и perimeter.
    return 
}
```

#### 3. Тип функции. Функция как параметр и результат функции
Раз функция — это полноценный тип данных, мы можем передавать её в другие функции. 

*Пример из жизни:* Представь, что ты вызываешь клининг. Ты можешь дать уборщику четкую инструкцию "помой пол" (вызвать конкретную функцию). А можешь передать ему чек-лист в виде конверта со списком действий (передать функцию как параметр). Уборщик просто вскроет конверт и выполнит то, что там написано.

```go
// Функция, которая принимает два числа и возвращает число
type MathOperation func(int, int) int 

func executeOperation(a, b int, op MathOperation) int {
    // Внутри мы не знаем, что делает op (складывает, вычитает?). Мы просто её вызываем.
    return op(a, b) 
}

func sum(x, y int) int { return x + y }
func sub(x, y int) int { return x - y }

func main() {
    // Передаем функцию sum как аргумент
    result1 := executeOperation(10, 5, sum) 
    result2 := executeOperation(10, 5, sub)
    fmt.Println(result1, result2) // Выведет: 15 5
}
```

#### 4. Анонимные функции
Часто нам нужна небольшая одноразовая функция, которую нет смысла называть и выносить отдельно. Мы можем создать её прямо на месте выполнения.

```go
func main() {
    // Создаем функцию, присваиваем её переменной и сразу используем
    greet := func(name string) {
        fmt.Printf("Привет, %s!\n", name)
    }
    greet("Дмитрий")

    // Анонимную функцию можно объявить и ВЫЗВАТЬ немедленно (IIFE - Immediately Invoked Function Expression)
    func(message string) {
        fmt.Println("Срочное сообщение:", message)
    }("Сервер запущен") // Скобки в конце вызывают эту безымянную функцию
}
```

#### 5. Замыкания (Closures)
Это концепция, которая часто взрывает мозг новичкам, но она невероятно мощная. Замыкание — это анонимная функция, которая "захватывает" переменные из области видимости, в которой она была создана.

*Пример из игровой логики:* Представь, что нам нужно считать серию убийств (kill streak) или комбо-удары, но мы не хотим делать счетчик глобальной переменной, чтобы его никто случайно не сломал из другой части кода. 

```go
// Функция возвращает другую функцию!
func newKillStreakCounter() func() int {
    streak := 0 // Эта переменная "закрыта" внутри создаваемой функции
    
    // Возвращаем анонимную функцию
    return func() int {
        streak++ // Функция помнит и имеет доступ к переменной streak
        return streak
    }
}

func main() {
    // counter1 - это теперь функция. При её создании streak инициализировался нулем.
    counter1 := newKillStreakCounter()
    
    fmt.Println("Kill streak:", counter1()) // Выведет 1
    fmt.Println("Kill streak:", counter1()) // Выведет 2
    fmt.Println("Kill streak:", counter1()) // Выведет 3
    
    // Если мы создадим новый счетчик, у него будет СВОЙ изолированный streak
    counter2 := newKillStreakCounter()
    fmt.Println("Новый игрок streak:", counter2()) // Выведет 1
}
```
Анонимная функция упаковала переменную `streak` в свой "рюкзак" и носит её с собой, даже когда оригинальная функция `newKillStreakCounter` уже завершила свою работу.

#### 6. Рекурсивные функции
Рекурсия — это когда функция вызывает саму себя для решения подзадачи.
Обязательное правило рекурсии — **условие выхода** (базовый случай), иначе функция будет вызывать себя бесконечно, пока не переполнит стек вызовов (Stack Overflow) и программа не упадет.

Классический пример — вычисление факториала. Математическая формула: n! = n × (n-1)!.



```go
func factorial(n int) int {
    // Базовый случай (условие выхода)
    if n == 0 {
        return 1
    }
    // Рекурсивный вызов: умножаем текущее число на факториал числа на единицу меньше
    return n * factorial(n - 1)
}

func main() {
    fmt.Println("Факториал 5:", factorial(5)) // 5 * 4 * 3 * 2 * 1 = 120
}
```
