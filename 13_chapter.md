### Глава 13. Базы данных

#### 1. Пакет `database/sql` и Архитектура Драйверов
Пакет `database/sql` из стандартной библиотеки — это не реализация работы с конкретной базой, а **интерфейс** (абстракция). Он предоставляет универсальные методы (`Query`, `Exec`), но сам по себе не знает, как говорить по сети с PostgreSQL или MySQL.

Для этого нужны **Драйверы** — сторонние библиотеки, которые реализуют этот интерфейс под капотом. 

*Пример из жизни:* Пакет `database/sql` — это универсальный пульт управления телевизором. А драйвер — это специальный чип-переходник, который переводит сигналы пульта на язык конкретного телевизора (Samsung, LG, Sony).

```go
package main

import (
    "database/sql"
    "fmt"
    "log"

    // Бланк-импорт (blank import). 
    // Мы ставим "_", чтобы компилятор не ругался на неиспользуемый пакет.
    // Нам не нужны функции из этого пакета напрямую, нам нужно, чтобы при 
    // инициализации пакета он молча зарегистрировал себя в database/sql как драйвер.
    _ "github.com/lib/pq" 
)

func main() {
    dsn := "host=localhost port=5432 user=postgres password=secret dbname=mydb sslmode=disable"
    
    // Функция Open НЕ устанавливает физическое соединение с БД!
    // Она лишь валидирует DSN (строку подключения) и готовит внутренние структуры.
    db, err := sql.Open("postgres", dsn)
    if err != nil {
        log.Fatal(err)
    }
    defer db.Close() // Закрываем пул при завершении программы

    // Чтобы реально проверить, жива ли база, используется Ping
    if err := db.Ping(); err != nil {
        log.Fatal("База недоступна:", err)
    }
    fmt.Println("Успешное подключение!")
}
```

#### 2. Управление пулом соединений (Connection Pooling)
Это **критически важная тема** для высоконагруженных систем.
Установка нового TCP-соединения с базой данных (особенно с SSL-хэндшейком и авторизацией) — это очень "дорогая" и медленная операция (десятки или сотни миллисекунд). 



Если на каждый HTTP-запрос мы будем открывать новое соединение, база данных быстро упадет от нехватки ресурсов (Connections Exhaustion). Go решает это с помощью встроенного **Пула Соединений**.

`*sql.DB` — это не одно соединение, это умный пул.
*Пример из жизни:* Пул — это таксопарк. Когда клиенту (горутине) нужно поехать (сделать SQL-запрос), он берет свободную машину. Когда поездка окончена, машина не уничтожается, а возвращается в парк и ждет следующего клиента.

**Настройка пула (Обязательно для Production):**
```go
// 1. Максимальное количество ОТКРЫТЫХ соединений (одновременно работающих машин)
// Защищает саму базу от перегрузки (например, в Postgres по умолчанию лимит 100).
db.SetMaxOpenConns(50) 

// 2. Максимальное количество ПРОСТАИВАЮЩИХ соединений (машин в ожидании на парковке).
// Должно быть близко к MaxOpenConns, чтобы при резком скачке трафика 
// нам не пришлось экстренно открывать новые коннекты.
db.SetMaxIdleConns(50)

// 3. Максимальное время жизни одного соединения.
// Балансировщики нагрузки и файрволы часто принудительно обрывают старые TCP-соединения.
// Если Go об этом не узнает, он попытается отправить запрос в "мертвый" коннект.
// Заставляем Go обновлять коннекты каждые 30 минут.
db.SetConnMaxLifetime(30 * time.Minute)
```

#### 3. Работа с реляционными БД (PostgreSQL, MySQL, SQLite)
Интерфейс работы одинаков для всех реляционных баз. Разница лишь в драйвере и диалекте SQL.

**Чтение одной строки (`QueryRow`):**
```go
var name string
var age int

// Знак доллара $1 - это плейсхолдер для защиты от SQL-инъекций (в MySQL используется '?').
// QueryRow сам возвращает соединение в пул после вызова Scan().
err := db.QueryRow("SELECT name, age FROM users WHERE id = $1", 1).Scan(&name, &age)
if err != nil {
    if err == sql.ErrNoRows {
        fmt.Println("Пользователь не найден")
    } else {
        log.Fatal(err)
    }
}
```

**Чтение множества строк (`Query`) и главная ловушка с памятью:**
```go
rows, err := db.Query("SELECT name, age FROM users WHERE age > $1", 18)
if err != nil {
    log.Fatal(err)
}
// ВАЖНО: Если не закрыть rows, TCP-соединение НИКОГДА не вернется в пул.
// Это самая частая причина утечки памяти и зависания серверов на Go!
defer rows.Close() 

for rows.Next() {
    err := rows.Scan(&name, &age)
    if err != nil {
        log.Fatal(err)
    }
    fmt.Printf("User: %s, Age: %d\n", name, age)
}

// Всегда проверяйте ошибки после цикла for rows.Next()
if err = rows.Err(); err != nil {
    log.Fatal(err)
}
```

**Запись данных (`Exec`):**
Используется для `INSERT`, `UPDATE`, `DELETE`, где нам не нужны возвращаемые строки данных.
```go
result, err := db.Exec("UPDATE users SET age = age + 1 WHERE id = $1", 1)
if err == nil {
    affected, _ := result.RowsAffected()
    fmt.Printf("Обновлено %d строк\n", affected)
}
```

#### 4. Транзакции и уровни изоляции
Транзакция гарантирует, что серия запросов выполнится целиком ("всё или ничего"). Если где-то произошла ошибка, мы обязаны сделать "откат" (`Rollback`), иначе изменения частично запишутся в базу.

```go
func transferMoney(db *sql.DB, fromID, toID int, amount float64) error {
    // Начинаем транзакцию. tx резервирует ОДНО жесткое соединение из пула.
    tx, err := db.Begin()
    if err != nil {
        return err
    }
    
    // Идиоматичный defer для отката в случае паники или забытого return
    defer tx.Rollback() 

    // Списываем деньги
    _, err = tx.Exec("UPDATE accounts SET balance = balance - $1 WHERE id = $2", amount, fromID)
    if err != nil {
        return err // При возврате сработает defer tx.Rollback()
    }

    // Зачисляем деньги
    _, err = tx.Exec("UPDATE accounts SET balance = balance + $1 WHERE id = $2", amount, toID)
    if err != nil {
        return err
    }

    // Если всё прошло успешно, фиксируем изменения.
    // После Commit(), вызов tx.Rollback() в defer просто безопасно игнорируется.
    return tx.Commit() 
}
```

#### 5. NoSQL (На примере MongoDB)
Так как `database/sql` спроектирован строго для табличных реляционных БД, для работы с NoSQL используются отдельные официальные драйверы. Например, `go.mongodb.org/mongo-driver/mongo`.

Там нет SQL-запросов. Данные передаются в виде структур BSON (бинарный JSON), что делает работу очень похожей на работу с мапами (хэш-таблицами) и структурами внутри самого языка.

```go
// Пример фильтра в MongoDB на Go
filter := bson.D{{"age", bson.D{{"$gt", 18}}}} // age > 18
cursor, err := collection.Find(context.TODO(), filter)
```
