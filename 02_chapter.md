### Глава 2. Основы языка Go

#### 1. Переменные и "Zero Values" (Нулевые значения)
В Go строгая статическая типизация. Это значит, что если ты создал коробку для целых чисел, положить туда строку (текст) уже не выйдет — компилятор ударит по рукам на этапе сборки. 

**Главное отличие Go от многих языков (например, C/C++)**: здесь нет "мусора" в неинициализированных переменных. 
*Пример из жизни:* Когда ты объявляешь переменную, Go ведет себя как ответственный арендодатель. Он не сдает тебе квартиру с хламом от прошлых жильцов (случайными байтами в оперативной памяти). Он делает генеральную уборку и присваивает переменной **Zero Value** (нулевое значение): для чисел это `0`, для строк — `""` (пустая строка), для булевых значений — `false`.

```go
package main

import "fmt"

func main() {
    // Способ 1: Явное объявление (Full declaration)
    // Используется, когда нужно объявить переменную сейчас, а присвоить значение позже
    var age int
    fmt.Println(age) // Выведет 0, так как сработало Zero Value

    // Способ 2: Объявление с инициализацией
    var name string = "Дмитрий"

    // Способ 3: Короткое объявление (Short variable declaration) `:=`
    // Go сам понимает (выводит) тип по значению. Это самый частый способ в коде.
    // Важно: `:=` работает ТОЛЬКО внутри функций.
    isDeveloper := true // Go понял, что это тип bool (логический)

    fmt.Println(name, isDeveloper)
}
```

#### 2. Базовые типы данных
Go дает разработчику полный контроль над тем, сколько памяти занимает переменная.
* **Числа целые:** `int8`, `int16`, `int32`, `int64` (цифра — это количество бит). Тип `int` — платформозависимый (на 64-битной ОС он будет равен `int64`). Беззнаковые: `uint8`, `uint16` и т.д.
* **Числа с плавающей точкой:** `float32`, `float64`.
* **Текст и символы:** `string` (неизменяемая строка байтов), `rune` (алиас для `int32`, представляет один Unicode-символ), `byte` (алиас для `uint8`).
* **Логический:** `bool` (`true` или `false`).

*Важно:* В Go нет неявного приведения типов. Нельзя просто сложить `int32` и `int64`, нужно явно конвертировать один тип в другой: `int64(myInt32)`. Это защищает от скрытых багов потери точности.

#### 3. Константы (`const`)
Константы вычисляются на этапе компиляции, а не во время работы программы (runtime). 
*Пример из жизни:* Переменная — это надпись маркером на доске (можно стереть и переписать). Константа — это гравировка на камне. Как только программа скомпилировалась, изменить её нельзя.

Особая фишка Go — генератор `iota` для создания перечислений (enum-подобных структур).

```go
const Pi float64 = 3.1415

const (
    // iota начинается с 0 и автоматически увеличивается на 1 для каждой следующей строки
    ReadMode = iota  // 0
    WriteMode        // 1 (тип и значение iota копируются автоматически)
    ExecuteMode      // 2
)
```

#### 4. Вывод на консоль и Форматирование (`fmt`)
Пакет `fmt` работает через так называемые "глаголы" (verbs) форматирования при использовании `Printf`.

```go
age := 30
name := "Alex"
// %s - строка, %d - целое число (digit)
// \n - перенос на новую строку
fmt.Printf("Имя: %s, Возраст: %d\n", name, age)

// Универсальные глаголы спасают при отладке:
// %v (value) - выводит значение в стандартном виде
// %+v - выводит структуру вместе с именами полей
// %T (type) - выводит тип переменной
fmt.Printf("Тип: %T, Значение: %v\n", name, name) 
```

#### 5. Операции: Арифметика, Присвоение, Поразрядные
* **Арифметика:** Стандартные `+`, `-`, `*`, `/`, `%` (остаток от деления).
* **Присвоение:** `=`, `+=`, `-=`, `*=` и т.д.
* **Поразрядные (Bitwise):** `&` (И), `|` (ИЛИ), `^` (Исключающее ИЛИ/XOR), `<<` (Сдвиг влево), `>>` (Сдвиг вправо). 
    *Они нужны для низкоуровневой работы.* Например, сдвиг влево `1 << 3` это по сути умножение 1 на $2^3$. Это работает гораздо быстрее для процессора, чем стандартное умножение.

#### 6. Массивы (Arrays)
В Go массив — это строго фиксированная последовательность элементов одного типа.
*Пример из жизни:* Массив — это коробка для таблеток на 7 дней. Ты не можешь добавить туда 8-ю ячейку. И что очень важно: в Go массив — это **значение**. Если ты передаешь массив в функцию, Go скопирует **все** его элементы, а не просто передаст адрес начала, как это делает C.



```go
// Объявление массива из 5 целых чисел. Все они инициализированы нулями.
var scores [5]int 

// Инициализация сразу со значениями
colors := [3]string{"red", "green", "blue"}

fmt.Println(scores[0]) // Доступ по индексу (начинается с 0)
```
*(В реальной разработке обычные массивы используются редко. Почти всегда используют Срезы (Slices), которые мы разберем в Главе 5, так как они умеют динамически расширяться).*

#### 7. Условные конструкции (`if` / `switch`)
**If / Else:** Скобки вокруг условия не нужны, а вот фигурные скобки `{}` обязательны всегда (никаких однострочников, как в C/Java).
Киллер-фича Go — **If с блоком инициализации**. Это позволяет создавать переменные, которые живут только внутри этого `if`.

```go
// Сначала вызываем функцию (например, получаем пользователя и ошибку).
// Затем ставим точку с запятой `;` и пишем само условие.
// Переменные user и err перестанут существовать за пределами фигурных скобок!
if user, err := getUser(1); err == nil {
    fmt.Println("Пользователь найден:", user)
} else {
    fmt.Println("Ошибка:", err)
}
```

**Switch:** В Go `switch` более безопасный. Он автоматически выходит из блока после совпадения (не нужно писать `break` в каждом `case`, как в других языках).

```go
day := "Tuesday"
switch day {
case "Saturday", "Sunday": // Можно перечислять через запятую
    fmt.Println("Выходной")
case "Monday", "Tuesday":
    fmt.Println("Будни")
default:
    fmt.Println("Неизвестный день")
}
```

#### 8. Циклы (`for`)
Go придерживается минимализма: здесь **нет** циклов `while` или `do-while`. Есть только `for`, но он умеет принимать разные формы.

```go
// 1. Классический цикл (инициализация; условие; шаг)
for i := 0; i < 5; i++ {
    fmt.Println(i)
}

// 2. Аналог цикла "while" (только условие)
counter := 0
for counter < 3 {
    fmt.Println(counter)
    counter++
}

// 3. Бесконечный цикл (часто используется в серверах для прослушивания соединений)
// Выйти из него можно с помощью ключевого слова 'break'
for {
    // Какая-то работа...
    break 
}
```
